# Вайб — История изменений

Сравнение проекта **до** и **после** серии улучшений.

---

## До

**8 файлов / ~2 400 строк кода**

```
index.html
pricing.html
profile.html  
profile.js
script.js
script-pricing.js
background.js
style.css
```

### Что было:
- Главная страница с анимацией, тремя карточками отзывов и формой обратной связи
- Страница тарифов (без обработчиков кнопок)
- Профиль пользователя с подпиской и аватаркой
- Firebase Auth + Firestore — базовая авторизация
- Отзывы открывались в модалке прямо на главной, отдельной страницы не было
- Кастомный курсор и частицы на фоне
- Логотип — просто текст без анимации
- Кнопка "Отзывы" в хедере открывала модалку на той же странице
- Нет страницы скачивания
- Нет страницы 404
- Нет мобильного адаптива для большинства блоков
- Один и тот же код авторизации в каждом файле
- Тема/настройки только на некоторых страницах
- При входе не проверялся `emailVerified` на главной

## Что изменилось

### Новые страницы

| Страница | Описание |
|---|---|
| `reviews.html` | Полноценная страница отзывов с фильтрами, сортировкой, пагинацией, лайками и ответами |
| `download.html` | Страница скачивания с карточками платформ, системными требованиями, секцией превью (видео + скриншоты), FAQ и changelog |
| `404.html` | Кастомная страница ошибки с обратным отсчётом, плавающими числами и кольцами |

### Архитектура

- **`firebase.js`** выделен в общий модуль — `showNotification`, `getPlaceholderDataURL`, `toggleDropdownMenu`, анимации входа/выхода, кастомный confirm-диалог
- **`auth-ui.js`** — единый модуль авторизации. Раньше один и тот же код (вход, регистрация, выход, настройки) дублировался в каждом файле, теперь выносится через `initAuth()`
- **Firestore rules** — добавлены правила для лайков (любой авторизованный) и ответов на отзывы (только администратор)

### Отзывы

- Отдельная страница с сеткой карточек вместо модалки на главной
- Фильтрация по количеству звёзд
- Сортировка: новые / популярные
- Пагинация с кнопкой «Загрузить ещё»
- **Сохранение позиции скролла** через `sessionStorage` — при возврате на страницу скролл восстанавливается
- Система лайков с оптимистичным обновлением UI
- Ответы администратора на отзывы
- Попап полного текста с анимацией **из точки нажатия** кнопки «читать далее»
- Loading skeleton с shimmer-анимацией при загрузке
- Статистика: количество отзывов, средняя оценка, доля пятёрок
- Динамический favicon с badge-счётчиком на вкладке браузера
- Real-time уведомления через `onSnapshot` — тост при новом отзыве

### Профиль

- История активности пользователя (отзывы, дата регистрации)
- **Кроппер аватарки** — круглое превью 200×200, перетаскивание мышью и тачем, зум слайдером и колёсиком, авто-масштаб при загрузке

### UI / UX

- **Анимированный логотип** — волна по буквам при наведении, светящаяся точка
- **Плавные переходы между страницами** — fade + slide при уходе и появлении
- **Счётчик символов** в форме отзыва на всех страницах
- Настройки (тема, частицы) теперь работают на всех страницах включая главную
- Дропдаун закрывается мгновенно при выходе без визуального артефакта
- Кастомный confirm-диалог вместо браузерного `window.confirm`

### Безопасность

- Проверка `emailVerified` при входе на всех страницах без исключения
- Firestore rules: лайки, ответы, валидация полей

### Мобильный адаптив

- Сетка тарифов: 3 колонки → 2 → 1 в зависимости от ширины экрана
- Сетка отзывов: 3 → 2 → 1
- Кнопки скачивания на всю ширину на мобилке
- Кастомный курсор отключается на touch-устройствах
- Аксерометр-параллакс на мобилках

### PWA

- `manifest.json` с иконками, shortcuts, описанием
- Service Worker с Network-First стратегией
- Firebase/CDN трафик не кэшируется, статика кэшируется для офлайн-работы
- Иконки 192×512 для Android
- `apple-touch-icon` для iOS
- Сайт можно «установить» на телефон через браузер
